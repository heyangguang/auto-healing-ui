# 代码仓库 (Git Repos) 功能指南

代码仓库模块用于管理 Ansible Playbook 的**源代码同步**。平台通过 Git 协议（HTTPS/SSH）连接远程仓库，自动克隆并定期拉取最新代码，确保 Playbook 库始终保持更新。

---

## 一、核心概念

### 1.1 仓库 (Repository)
一个代码仓库对应一个远程 Git 仓库地址。仓库被克隆到平台本地存储后，平台会自动扫描其中的 Ansible Playbook 文件（`.yml` / `.yaml`），生成对应的 Playbook 记录。

### 1.2 同步机制
- **首次同步**：执行 `git clone`，全量克隆仓库到本地。
- **后续同步**：执行 `git pull`，增量拉取变更并重新扫描 Playbook。
- **定时同步**：开启后会按配置的间隔（如 `1h`）自动触发同步。

---

## 二、仓库状态 (status) 字典

| 枚举值 | 中文含义 | 说明 |
|--------|---------|------|
| `syncing` | **同步中** | 正在执行 clone 或 pull 操作 |
| `ready` | **就绪** | 同步成功，仓库可正常使用 ✅ |
| `error` | **错误** | 同步失败（如认证错误、网络不通） ❌ |

### 同步日志状态

| 枚举值 | 中文含义 | 动作类型 |
|--------|---------|---------|
| `success` | 成功 | `clone`（首次同步）/ `pull`（增量同步） |
| `failed` | 失败 | 包含详细的错误信息 |

### 触发方式 (trigger_type)

| 枚举值 | 中文含义 |
|--------|---------|
| `manual` | 手动触发 |
| `scheduled` | 定时自动触发 |

---

## 三、认证方式 (auth_type)

| 枚举值 | 中文含义 | 适用场景 |
|--------|---------|---------|
| `none` | **无认证** | 公开仓库 |
| `token` | **Personal Access Token** | GitHub/GitLab Token 认证 |
| `password` | **用户名密码** | HTTPS + 用户名密码 |
| `ssh_key` | **SSH 密钥** | SSH 协议 + 私钥认证 |

---

## 四、字段说明

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **名称** | 仓库的显示名称 | `生产级自愈Playbook` |
| **URL** | Git 远程仓库地址 | `https://github.com/ops/playbooks.git` |
| **默认分支** | 拉取的目标分支 | `main` |
| **认证方式** | 连接仓库的认证类型 | `token` |
| **状态** | 当前仓库状态 | `ready` |
| **Playbook 数量** | 扫描到的 Playbook 数量 | `15` |
| **同步间隔** | 定时同步的间隔时间 | `1h` |
| **上次同步** | 最后一次成功同步时间 | `10 分钟前` |
| **最新 Commit** | 最后同步到的 commit SHA（前 8 位） | `a3b8d1b6` |

## 五、核心操作

### 5.1 添加仓库
1. 点击 **+ 新建** → 填写仓库名称和 Git URL。
2. 选择认证方式并填写凭据。
3. 选择默认分支（默认 `main`）。
4. 可选开启定时同步并设置间隔。
5. 保存后平台立即开始首次 clone 操作。

### 5.2 手动同步
在操作栏点击 **同步** 按钮可立即触发一次 pull 操作。

### 5.3 查看同步日志
点击 **日志** 图标可浏览该仓库的完整同步历史，包括每次的耗时、commit ID、触发方式等。

### 5.4 删除仓库
删除仓库会同时**删除该仓库下扫描到的所有 Playbook 记录**。已被任务模板引用的 Playbook 无法删除。

---

## 六、自动暂停机制

与插件类似，仓库也支持连续失败自动暂停：
- 当连续同步失败达到 `max_failures`（默认 5 次）时，定时同步自动暂停。
- 管理员需排查错误原因（如 Token 过期、仓库地址变更）后手动恢复。

---

## 七、最佳实践

1. **仓库结构**：建议按 `playbooks/{功能}/*.yml` 组织 Playbook 文件，便于扫描和管理。
2. **使用 Token 认证**：Token 方式比密码更安全，且易于轮转和权限控制。
3. **合理同步间隔**：活跃开发仓库设 10-30 分钟，稳定仓库可设 1-6 小时。
4. **分支策略**：生产环境使用 `main`/`release` 分支，避免同步开发中的不稳定代码。
