# 快速创建自愈流程

从零开始配置一个完整的自动化运维流程。

## 什么是自愈流程？

自愈流程是 Auto-Healing 平台的核心能力。当监控系统或外部插件检测到故障告警时，平台会根据预定义的规则自动匹配对应的执行任务，在无人值守的情况下完成故障修复。整个流程由可视化 DAG 编辑器编排，支持条件分支、并行执行和人工审批节点。

## 前置准备

在创建自愈流程之前，你需要：

1. **至少一台纳管主机** — 确保主机已添加到 CMDB 并处于在线状态
2. **至少一个 Playbook** — 可以从 Git 仓库自动同步，也可以手动编写
3. **对应的 SSH 密钥** — 在密钥管理中添加用于连接目标主机的凭证

这三项是自愈流程运行的基础设施。

## 创建步骤详解

### 第一步：纳管主机

进入 **资源配置 > 资产管理**，点击「添加主机」。填写主机 IP、SSH 端口、选择认证密钥，添加后平台会自动检测主机连通性。建议将相同用途的主机归入同一分组，方便后续批量操作。

### 第二步：导入 Playbook

进入 **作业中心 > 代码仓库**，添加你的 Git 仓库地址。仓库同步完成后，系统会自动扫描仓库中的 Playbook 文件并注册到剧本管理。你也可以手动在 **剧本管理** 页面导入单个 Playbook。

### 第三步：创建任务模板

进入 **作业中心 > 任务模板**，点击「创建模板」。选择要关联的 Playbook、配置默认运行参数（extra vars）、选择目标主机组。任务模板是自愈规则的最小执行单元。

### 第四步：配置自愈规则

进入 **自愈引擎 > 自愈规则**，点击「新建规则」。配置触发条件（如插件名称、告警级别、关键词匹配），选择要执行的任务模板。规则支持优先级排序和条件组合。

### 第五步：启用自愈流程

进入 **自愈引擎 > 自愈流程**，点击「新建流程」进入 DAG 编辑器。将规则节点、任务节点、审批节点拖入画布，连线编排执行顺序。保存后启用流程，即可开始自动化自愈。

## 最佳实践

- **先 Dry Run 再生产** — 新规则建议先在测试主机上启用模拟运行（Dry Run），确认无误后再切换到正式模式
- **设置审批节点** — 对于高危操作（如重启服务），建议在流程中插入人工审批节点
- **配置通知** — 为自愈规则绑定通知模板，确保团队能收到执行结果的反馈
- **分级处理** — 将规则按优先级分级：P1（自动执行）、P2（审批后执行）、P3（仅通知）
