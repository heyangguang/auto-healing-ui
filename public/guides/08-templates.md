# 任务模板 (Templates) 功能指南

![任务模板列表](/guides/images/execution_templates.webp)

任务模板是 Playbook 的**可执行封装**。它在 Playbook 的基础上绑定了具体的目标主机、变量值、执行器配置和通知策略，使得一个抽象的 Playbook 脚本可以被一键触发执行。任务模板也是自愈流程中 `execution` 节点引用的执行对象。

---

## 一、核心概念

### 1.1 模板 vs Playbook
- **Playbook**：通用脚本（相当于"模具"）
- **任务模板**：绑定了具体参数的可执行实例（相当于"成品"）

例如：一个"清理磁盘"的 Playbook 可以创建多个模板：
- 模板 A：清理 Web 服务器组的 /tmp 目录
- 模板 B：清理数据库服务器组的 /var/log 目录

---

## 二、字段详解

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **名称** (name) | 模板名称 | `日志归档清理-应用服务器` |
| **描述** (description) | 模板用途说明 | `每日清理应用服务器的日志文件` |
| **关联 Playbook** (playbook_id) | 引用的 Playbook | `disk-cleanup-playbook` |
| **目标主机** (target_hosts) | 执行目标 | `webserver-group` 或 `10.0.1.1,10.0.1.2` |
| **变量值** (extra_vars) | Playbook 变量的具体值 | `{"path":"/tmp","days":7}` |
| **执行器类型** (executor_type) | 执行环境 | `local` / `docker` |
| **密钥源** (secrets_source_ids) | 认证使用的密钥 | 引用密钥管理中的密钥源 |
| **需要审核** (needs_review) | 是否有变量变更待确认 | `true` / `false` |
| **定时任务数** (schedule_count) | 关联的定时任务数量 | `3` |

### 执行器类型 (executor_type) 字典

| 枚举值 | 中文含义 | 说明 |
|--------|---------|------|
| `local` | **本地执行** | 直接在平台服务器上通过 SSH 执行 Ansible |
| `docker` | **Docker 容器** | 在隔离的 Docker 容器中执行，更安全 |

---

## 三、通知配置

任务模板支持配置执行完成后的自动通知：

| 配置项 | 说明 |
|--------|------|
| **启用通知** (enabled) | 是否在执行完成后发送通知 |
| **通知渠道** | 选择已配置的通知渠道（邮件/钉钉/Webhook） |
| **通知模板** | 选择通知内容模板 |
| **触发条件** | 成功时通知 / 失败时通知 / 始终通知 |

## 四、核心操作

### 4.1 创建任务模板
1. 点击 **+ 新建** → 填写名称，选择关联的 Playbook。
2. 配置目标主机：选择 CMDB 主机组或手动输入主机列表。
3. 填写变量值：基于 Playbook 声明的变量，填写具体的参数值。
4. 选择执行器类型和密钥源。
5. 可选配置通知规则。
6. 保存模板。

### 4.2 直接执行
在模板列表点击 **执行** 按钮可立即触发一次任务执行。执行结果会记录在**执行记录**中。

### 4.3 变量审核
当关联的 Playbook 变量发生变更时，模板会被标记为 `needs_review`。需要：
1. 在模板列表中找到标记为"需要审核"的模板。
2. 查看变更的变量详情。
3. 确认或调整变量值后解除审核标记。

### 4.4 编辑与删除
- **编辑**：修改任何配置项。变更不会影响已执行的历史记录。
- **删除**：有关联定时任务的模板**无法直接删除**。

---

## 五、最佳实践

1. **命名规范**：建议 `{操作}-{目标范围}` 格式，如 `日志清理-生产数据库`。
2. **Docker 执行器**：对于不信任的 Playbook 或多租户环境，推荐使用 Docker 执行器隔离风险。
3. **密钥绑定**：为每个模板显式绑定密钥源，避免依赖优先级自动匹配。
4. **通知配置**：至少为重要的生产模板配置失败通知。
