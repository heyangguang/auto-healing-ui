# 系统管理功能指南

![系统管理](/guides/images/system_users_roles.png)

系统管理模块提供平台级的**用户、角色、权限和审计**治理能力。通过 RBAC（基于角色的访问控制）模型，实现精细化的功能权限管理和完整的操作审计追溯。

---

## 一、用户管理

### 1.1 用户字段

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **用户名** (username) | 登录账号（唯一） | `zhangsan` |
| **显示名** (display_name) | 界面展示的姓名 | `张三` |
| **邮箱** (email) | 用户邮箱地址 | `zhangsan@company.com` |
| **角色** (role) | 分配的系统角色 | `运维人员` |
| **状态** (status) | 账号状态 | `active` / `locked` / `disabled` |
| **最后登录** (last_login_at) | 上次登录时间 | `2024-01-15 09:00:00` |
| **创建时间** (created_at) | 账号创建时间 | `2023-06-01` |

### 1.2 用户操作
- **新增用户**：创建新账号，分配角色，设置初始密码。
- **编辑用户**：修改显示名、邮箱、角色。
- **重置密码**：为用户重置登录密码。
- **锁定/解锁**：锁定账号禁止登录，解锁恢复访问。
- **删除用户**：移除用户账号。系统管理员 (admin) 账号不可删除。

## 二、角色管理

### 2.1 内置角色

| 角色 | 代码 | 说明 |
|------|------|------|
| **系统管理员** | `admin` | 拥有所有权限，不可删除不可修改 |
| **运维人员** | `operator` | 可执行任务、管理资产和流程 |
| **只读用户** | `viewer` | 仅查看权限，不可修改任何配置 |

### 2.2 自定义角色
支持创建自定义角色并分配细粒度权限：
1. 点击 **+ 新建** 创建角色。
2. 设置角色名称和描述。
3. 通过**权限分配器**（双列表穿梭框）选择该角色拥有的操作权限。
4. 保存。

### 2.3 权限粒度
权限按模块和操作类型组织：

| 模块 | 权限示例 |
|------|---------|
| CMDB | 查看资产、创建资产、编辑资产、删除资产 |
| 执行任务 | 查看模板、创建模板、执行任务、查看日志 |
| 自愈管理 | 查看流程、创建流程、管理规则、审批操作 |
| 系统管理 | 用户管理、角色管理、审计日志、租户管理 |

## 三、租户管理

平台支持**多租户**模式。每个租户拥有独立的数据空间和资源隔离。

### 3.1 租户字段

| 字段 | 说明 |
|------|------|
| **名称** (name) | 租户显示名称 |
| **编码** (code) | 租户唯一编码标识 |
| **状态** | 启用/停用 |
| **用户数** | 租户下的用户数量 |

### 3.2 租户切换
用户可通过顶部导航栏的**租户切换器**在已授权的租户之间快速切换。切换后数据会自动刷新为目标租户的数据。

## 四、审计日志

### 4.1 日志字段

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **操作人** (username) | 执行操作的用户 | `admin` |
| **操作类型** (action) | 操作类型 | 见下表 |
| **资源类型** (resource_type) | 被操作的资源类型 | 见下表 |
| **资源名称** (resource_name) | 被操作的具体资源 | `磁盘清理模板` |
| **状态** (status) | 操作是否成功 | `success` / `failed` |
| **IP 地址** (ip_address) | 操作人的来源 IP | `192.168.1.100` |
| **时间** (created_at) | 操作时间 | `2024-01-15 10:30:00` |

### 4.2 操作类型 (action) 字典

| 枚举值 | 中文含义 |
|--------|---------|
| `create` | **创建** |
| `update` | **修改** |
| `delete` | **删除** |
| `enable` | **启用** |
| `disable` | **停用** |
| `execute` | **执行** |
| `login` | **登录** |
| `export` | **导出** |

### 4.3 资源类型 (resource_type) 字典

| 枚举值 | 中文含义 |
|--------|---------|
| `healing_flow` | 自愈流程 |
| `healing_rule` | 自愈规则 |
| `execution_task` | 执行任务 |
| `execution_schedule` | 定时任务 |
| `playbook` | 剧本 |
| `notification_template` | 通知模板 |
| `notification_channel` | 通知渠道 |
| `cmdb` | 主机 |
| `secret` | 密钥 |
| `user` | 用户 |
| `role` | 角色 |
| `tenant` | 租户 |
| `git_repo` | Git 仓库 |
| `plugin` | 插件 |

### 4.4 审计日志功能
- **搜索**：按操作人、资源名称搜索
- **筛选**：按操作类型、资源类型、时间范围筛选
- **导出**：支持将审计日志导出为 CSV 文件
- **排除过滤**：可排除特定操作类型（如排除 login 只看业务操作）

## 五、最佳实践

1. **最小权限原则**：为用户分配满足工作需要的最小权限集，避免过度授权。
2. **定期审计**：每月审查审计日志，检查是否有异常操作。
3. **角色复用**：通过自定义角色实现权限复用，避免逐用户配置。
4. **密码策略**：建议定期更换密码，管理员账号应使用强密码。
