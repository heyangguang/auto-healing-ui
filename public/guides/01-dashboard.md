# 仪表盘 (Dashboard) 功能指南

![仪表盘总览](/guides/images/dashboard_overview.webp)

仪表盘是 Auto Healing 系统的**全局监控与态势总览中心**，是登录后看到的第一个页面。它将系统中分散在各模块的关键指标与状态信息聚合到一个界面中，帮助管理员和运维人员**一眼掌握**全局运行态势，快速定位异常或需要关注的热点。

> **适用场景：** 每日晨检、值班交接、故障快速定位、SLA 报告采集。

---

## 一、运维统计面板

仪表盘顶部展示一组核心运营指标卡片，每张卡片聚焦一个维度的实时数据：

### 1.1 工单统计
- **今日新增工单数**：当日零点起累计收到的告警/工单总量。
- **待处理工单**：处于 `pending`（待触发）或 `in_progress`（处理中）状态的工单数量。点击后可直接跳转到 **工单列表** 进一步处理。
- **近 7 天趋势**：以数值形式展示近一周工单变化趋势。

### 1.2 自愈执行指标
- **今日成功 / 失败次数**：快速判断自愈引擎是否正常运转。
- **成功率**：展示自愈执行的成功百分比，帮助量化运维自动化的有效性。低于预设阈值时，建议立即检查 Playbook 和目标主机的状态。

### 1.3 纳管主机概览
- **在线主机数**：当前可达（SSH 连通性测试通过）的受控节点数量。
- **离线 / 维护**：离线不可达或人工设置维护窗口的主机数。
- 点击后直接跳转到 **CMDB 资产管理** 页面查看详情。

---

## 二、系统健康监控

系统健康卡片提供平台自身的运行状况指标：

| 指标 | 说明 | 典型值域 |
|------|------|---------|
| **运行状态** | 系统整体健康度：正常 / 降级 / 异常 | 🟢 正常 🟡 降级 🔴 异常 |
| **系统版本** | 当前部署的 Auto Healing 平台版本号 | 如 `v2.3.1` |
| **运行时间** | 系统自上次重启以来的持续运行时间 | 如 `15天 3小时` |
| **环境** | 当前部署环境标识 | `production` / `staging` |
| **API 响应** | 后端 API 接口的平均延迟（毫秒） | 正常应 < 100ms |
| **数据库** | PostgreSQL 数据库查询延迟（毫秒） | 正常应 < 50ms |

> **提示：** 当 API 响应延迟 > 500ms 或数据库延迟 > 200ms 时，建议检查服务器负载和数据库连接池状态。

---

## 三、待办审批

实时展示需要人工介入的待办事项，包括两类：

1. **待触发工单**：经自愈规则匹配但需手动确认才能触发执行的告警事件。
2. **待审批任务**：自愈流程中设置了人工审批节点，正在等待运维人员审核通过的执行步骤。

每条待办展示：
- 严重等级色标（🔴 严重 / 🟠 高 / 🟡 中 / ⚪ 低）
- 工单标题与影响的主机/CI
- 等待时间

点击 **查看全部** 可跳转到 **待办中心** 进行批量处理。

---

## 四、平台资源概览

以 4 × 2 网格形式展示平台纳管的各类资源总数：

| 资源类型 | 说明 |
|---------|------|
| 主机 | CMDB 中纳管的物理机/虚拟机/容器数量 |
| 剧本 | 已注册并扫描的 Ansible Playbook 数量 |
| 任务模板 | 封装好的可执行任务模板数量 |
| 自愈流程 | 已编排的 DAG 工作流数量 |
| 自愈规则 | 已配置的告警匹配规则数量 |
| 通知渠道 | 已配置的邮件/钉钉/Webhook 通道数量 |
| 定时任务 | 活跃的 Cron 定时调度数量 |
| 插件 | 已安装的外部系统集成插件数量 |

点击任意统计卡可直接导航到对应的管理页面。

---

## 五、变更记录

展示系统中最近的操作审计日志，包括：
- **操作人**：执行操作的用户姓名
- **操作类型**：创建、修改、删除、启用、禁用
- **影响对象**：被操作的资源名称和类型（如自愈流程、密钥、主机等）
- **时间**：操作发生的相对时间（如"3 分钟前"）

> **提示：** 完整的审计日志可在 **系统管理 → 审计日志** 中查看和导出。

---

## 六、定时任务日历

右侧栏的日历视图以**蓝色圆点**标记有定时任务的日期。选中某一天后，下方会展示该日所有计划执行的任务列表，包括：
- 执行时间（如 `02:00`）
- 任务名称
- 若同一任务在一天内执行多次，会自动合并显示次数标签（如 `48次/天`）

---

## 七、最佳实践

1. **每日晨检**：登录后先查看仪表盘，确认系统健康状态和待办事项数量。
2. **阈值告警**：当自愈成功率低于 90% 时，建议立即排查失败的自愈实例。
3. **资源监控**：保持关注纳管主机的在线率，离线主机过多可能意味着网络问题。
4. **审计追溯**：遇到异常变更时，优先在变更记录中查找最近的操作人和操作内容。
