# 通知模板配置

配置邮件、钉钉、Webhook 多渠道告警通知。

## 多渠道通知体系

平台支持多种通知渠道，确保告警信息能及时触达相关人员：

| 渠道类型 | 适用场景 | 配置要求 |
|---------|---------|--------|
| 邮件 (SMTP) | 正式通知、详细报告 | SMTP 服务器地址、端口、认证信息 |
| 钉钉机器人 | 即时告警、团队协作 | Webhook URL、签名密钥（可选） |
| 企业微信 | 企业内部通知 | Webhook URL |
| 自定义 Webhook | 与第三方系统集成 | 目标 URL、认证方式 |

每个渠道配置完成后，建议先发送测试消息验证连通性。

## 模板变量参考

通知模板支持以下内置变量：

**告警信息**：

- `{{ .event_name }}` — 告警/事件名称
- `{{ .severity }}` — 告警级别
- `{{ .description }}` — 告警描述
- `{{ .triggered_at }}` — 触发时间

**主机信息**：

- `{{ .host_ip }}` — 目标主机 IP
- `{{ .host_name }}` — 主机名称
- `{{ .host_group }}` — 主机分组

**执行结果**：

- `{{ .task_name }}` — 任务模板名称
- `{{ .execution_status }}` — 执行状态（成功/失败）
- `{{ .execution_duration }}` — 执行耗时
- `{{ .log_url }}` — 日志链接

## 通知最佳实践

- **分级通知** — 严重告警用即时通讯（钉钉/企微），普通事件用邮件汇总
- **避免告警风暴** — 平台内置滑动窗口限流策略，防止短时间内发送过多重复通知
- **模板复用** — 创建通用模板后，多个规则可以共享同一个通知模板
- **退避重试** — 通知发送失败时，平台会自动按指数退避策略重试
