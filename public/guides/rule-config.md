# 规则配置指南

如何配置触发条件、匹配策略和执行动作。

## 规则引擎概述

自愈规则是连接「告警事件」和「修复动作」的桥梁。当外部告警到达平台时，规则引擎会按优先级依次匹配所有启用的规则，找到第一个匹配的规则后执行对应的任务模板。

规则由三部分组成：

1. **触发条件** — 定义什么样的告警会激活这条规则
2. **匹配策略** — 如何从告警数据中提取关键信息
3. **执行动作** — 匹配成功后执行什么任务

## 触发条件配置

**插件匹配**：指定告警来源插件（如 Zabbix、Prometheus、ServiceNow），只处理特定来源的告警。

**字段匹配**：

- `severity` — 告警级别匹配（critical / high / medium / low）
- `alert_name` — 告警名称关键词匹配
- `cmdb_ci` — 受影响 CI 匹配（支持通配符）

**表达式匹配**：

使用 `expr` 表达式引擎，支持复杂的条件组合：

```
severity == "critical" && contains(alert_name, "disk") && affected_ci matches "web-*"
```

## 执行动作与参数映射

匹配成功后，规则会将告警数据中的字段映射为任务模板的执行参数：

- `{{ .alert.cmdb_ci }}` → 目标主机
- `{{ .alert.severity }}` → 告警级别
- 自定义字段 → Extra Vars

**执行模式**：

- **自动执行** — 匹配后立即执行任务，适用于低风险操作
- **审批执行** — 匹配后创建审批工单，经人工确认后执行
- **仅通知** — 不执行任务，仅发送通知提醒
