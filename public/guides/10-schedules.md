# 定时任务 (Schedules) 功能指南

定时任务模块提供基于 Cron 表达式的**周期性自动执行**能力。通过绑定任务模板，可以实现定时巡检、定期清理、周期性备份等无人值守的运维操作。

---

## 一、核心概念

### 1.1 定时任务 vs 任务模板
- **任务模板**：定义"做什么"——执行哪个 Playbook、目标主机、变量值。
- **定时任务**：定义"什么时候做"——Cron 时间表达式和启停控制。

一个任务模板可以关联多个定时任务（如工作日每天 2:00 AM 和周末每天 4:00 AM）。

---

## 二、字段详解

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **名称** (name) | 定时任务的描述性名称 | `每日清理-Web服务器日志` |
| **任务模板** (task_id) | 绑定的任务模板 | `日志归档清理-应用服务器` |
| **Cron 表达式** (cron_expression) | 标准 5 位 Cron 表达式 | `0 2 * * *`（每天凌晨 2 点） |
| **启用状态** (is_active) | 是否启用 | `true` / `false` |
| **上次执行** (last_run_at) | 上次触发执行的时间 | `2024-01-15 02:00:00` |
| **下次执行** (next_run_at) | 下次计划执行的时间 | `2024-01-16 02:00:00` |
| **说明** (description) | 简要备注 | `工作日每天凌晨清理 7 天前的日志` |

### Cron 表达式格式

```
┌───────── 分 (0-59)
│ ┌───────── 时 (0-23)
│ │ ┌───────── 日 (1-31)
│ │ │ ┌───────── 月 (1-12)
│ │ │ │ ┌───────── 周 (0-6, 0=周日)
│ │ │ │ │
* * * * *
```

常用示例：

| 表达式 | 含义 |
|--------|------|
| `0 2 * * *` | 每天凌晨 2:00 |
| `*/30 * * * *` | 每 30 分钟 |
| `0 9 * * 1-5` | 工作日每天 9:00 |
| `0 0 1 * *` | 每月 1 号零点 |
| `0 */6 * * *` | 每 6 小时 |

## 三、核心操作

### 3.1 创建定时任务
1. 点击 **+ 新建**，填写任务名称。
2. 选择要执行的任务模板。
3. 填写 Cron 表达式（提供可视化 Cron 编辑器）。
4. 选择是否立即启用。
5. 保存。

### 3.2 暂停与恢复
通过开关可以暂停和恢复定时任务。暂停时不会触发执行，但配置保留。

### 3.3 手动触发
点击 **立即执行** 可不等待 Cron 周期，立即触发一次执行。

### 3.4 查看执行历史
点击 **历史** 可以查看该定时任务的所有历史执行记录及结果。

---

## 四、在仪表盘中的展示

仪表盘右侧的**日历视图**会以蓝色圆点标记有定时任务的日期。选中特定日期后可预览当天所有计划执行的任务列表和时间。

---

## 五、最佳实践

1. **错峰执行**：避免多个定时任务在同一时间触发，分散到不同时段。
2. **先手动验证**：新创建的定时任务建议先手动触发一次确认无误，再启用 Cron。
3. **监控执行记录**：定期检查定时任务的历史执行结果，确保没有持续性失败。
4. **注意时区**：Cron 使用服务器时区（默认 UTC），配置时需注意时区差异。
