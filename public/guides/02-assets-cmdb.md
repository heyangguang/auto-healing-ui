# 资产管理 (CMDB) 功能指南

![CMDB 资产列表](/guides/images/cmdb_overview.webp)

资产管理模块（CMDB）是 Auto Healing 系统的**基础设施底座**。所有后续的 Playbook 执行、自愈流程触发、定时任务调度，都依赖于 CMDB 中注册的节点信息。没有在 CMDB 中注册的主机，无法成为任何任务的执行目标。

> **核心原则：** CMDB 是整个平台的 **Single Source of Truth (SSOT)**，所有操作的目标主机信息均以 CMDB 为准。

---

## 一、核心概念

### 1.1 资产节点 (Asset / Node)
资产节点代表一台可被 Auto Healing 平台管理和操作的计算资源。可以是物理服务器、虚拟机、容器或云实例。每个节点具有唯一的主机名标识。

### 1.2 主机分组 (Host Group)
主机可以通过**分组**进行逻辑组织（如按环境分为 `Production`、`Staging`；按业务分为 `WebServer`、`Database` 等）。分组用于：
- 任务模板批量绑定执行目标
- 权限隔离（不同角色可见不同分组）
- 自愈规则的目标范围筛选

### 1.3 维护模式 (Maintenance Mode)
当主机处于维护模式时，平台将**自动跳过**该主机的所有自愈任务和定时任务执行。这适用于计划内停机、升级窗口等场景，避免在维护期间触发不必要的自动化操作。

---

## 二、资产列表字段说明

平台的资产列表以表格形式展示所有纳管节点，支持的字段如下：

| 字段名 | 说明 | 示例值 |
|--------|------|--------|
| **名称** | 资产的用户自定义名称，支持搜索 | `web-server-01` |
| **主机名** | 系统实际主机名 / FQDN | `srv-prod-01.dc1.local` |
| **IP 地址** | SSH 连接的目标 IP | `10.0.1.100` |
| **SSH 端口** | SSH 服务监听端口 | 默认 `22` |
| **类型** | 节点类型分类 | `物理机` / `虚拟机` / `容器` |
| **操作系统** | 运行的操作系统 | `CentOS 7.9` / `Ubuntu 22.04` |
| **状态** | 当前状态 | 🟢 在线 / 🔴 离线 / 🟡 维护中 |
| **分组** | 所属的主机分组 | `Production-WebServer` |
| **负责人** | 该资产的责任人 | `ops-team-a` |
| **环境** | 部署环境标识 | `生产` / `测试` / `开发` |
| **最后检测** | 平台最近一次连通性检测的时间 | `2 分钟前` |

## 三、核心操作指南

### 3.1 手动添加主机

1. 点击列表上方的 **+ 新建** 按钮。
2. 在弹出的表单中填写基本信息：名称、主机名、IP 地址、SSH 端口。
3. 选择认证方式：
   - **密钥认证**：选择预先在密钥管理中配置的 SSH 私钥。
   - **密码认证**：直接输入用户名和密码。
4. 选择主机分组（可多选），设置环境标签。
5. 点击 **确定** 完成添加。平台会立即尝试 SSH 连通性测试。

> ⚠️ **注意：** 确保平台服务器到目标主机之间的网络是通的，且防火墙允许 SSH 端口访问。

### 3.2 批量导入

支持通过 **CSV 文件** 批量导入多台主机：
1. 点击 **导入** 按钮下载 CSV 模板。
2. 按模板格式填写主机信息（每行一台主机）。
3. 上传填写好的 CSV 文件，平台会校验数据格式。
4. 确认无误后执行导入，可在列表中查看导入结果。

### 3.3 编辑与删除

- **编辑**：点击操作列的编辑图标，修改主机的任意属性。修改 IP 或端口后会重新检测连通性。
- **删除**：选中一台或多台主机后点击批量删除。**已被任务模板引用的主机无法直接删除**，需先解除引用关系。

### 3.4 状态切换

通过操作列的状态开关，可以手动将主机切换为：
- **在线**：正常状态，参与所有自动化任务。
- **维护中**：进入维护模式，暂停所有自动化操作。

### 3.5 连通性检测

平台会定期（默认每 5 分钟）对所有纳管主机进行 SSH 连通性探测。也可以手动点击 **重新检测** 按钮立即触发检测。

### 3.6 插件同步

如果安装了 CMDB 类插件（如连接外部 CMDB 系统），主机数据可以通过定期同步自动更新，无需手动维护。

## 四、高级功能

### 4.1 主机详情抽屉

点击主机名称可打开右侧详情抽屉，展示该主机的完整信息：
- **基本信息面板**：IP、端口、操作系统、环境、负责人等。
- **硬件规格**：CPU 核心数、内存大小、磁盘容量。
- **执行历史**：该主机作为目标参与过的所有任务执行记录。
- **关联规则**：引用了该主机（或其所在分组）的自愈规则。

### 4.2 搜索与筛选

- **关键词搜索**：支持按名称、主机名、IP 等关键词快速查找。
- **高级筛选**：可按状态（在线/离线/维护）、类型、环境、分组等条件组合筛选。
- **排序**：支持按名称、状态、最后检测时间等字段排序。

---

## 五、最佳实践

1. **命名规范**：建议采用 `{环境}-{业务}-{序号}` 的命名格式，如 `prod-web-01`。
2. **分组策略**：按环境 × 业务交叉建组，如 `Production-Database`、`Staging-WebServer`。
3. **密钥管理**：为不同环境使用不同的 SSH 密钥，避免跨环境凭证混用。
4. **维护流程**：计划内维护时，提前将受影响主机切换到维护模式，维护结束后再恢复。
5. **定期审计**：每月检查是否有长期离线的主机需要清理或下线。
