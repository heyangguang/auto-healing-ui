# 自愈实例 (Healing Instances) 功能指南

![自愈实例列表](/guides/images/healing_instances.png)

自愈实例是自愈流程的**运行时快照和执行记录**。每当自愈规则匹配工单触发流程时，就会创建一个新的流程实例。实例忠实记录了流程从启动到终止的全过程，包括每个节点的状态变迁、执行日志、上下文数据和审批记录。

---

## 一、流程实例状态 (status) 字典

| 枚举值 | 中文含义 | 颜色标识 | 说明 |
|--------|---------|---------|------|
| `pending` | **待启动** | ⚪ 灰色 | 实例已创建但尚未开始执行 |
| `running` | **运行中** | 🔵 蓝色 | 流程正在执行节点逻辑 |
| `waiting_approval` | **等待审批** | 🟠 橙色 | 流程执行到审批节点，暂停等待人工决策 🕐 |
| `completed` | **已完成** | 🟢 绿色 | 流程成功到达 end 节点，正常结束 ✅ |
| `failed` | **失败** | 🔴 红色 | 流程执行过程中出现不可恢复的错误 ❌ |
| `cancelled` | **已取消** | ⚫ 黑色 | 用户手动取消了正在运行的流程实例 🚫 |

---

## 二、审批任务状态 (approval_status) 字典

当流程运行到审批节点时，会创建一条审批任务记录：

| 枚举值 | 中文含义 | 说明 |
|--------|---------|------|
| `pending` | **待审批** | 审批任务已创建，等待审批人处理 |
| `approved` | **已批准** | 审批人点击"通过"，流程继续执行 ✅ |
| `rejected` | **已拒绝** | 审批人点击"拒绝"，流程走拒绝分支 ❌ |
| `expired` | **已超时** | 超过设定的超时时间无人审批 ⏰ |

---

## 三、字段说明

### 3.1 实例概览

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **实例 ID** | 唯一标识 | `fi-2024-0115-001` |
| **流程名称** (flow_name) | 来源的流程模板名称 | `磁盘空间自愈流程` |
| **关联工单** (incident_id) | 触发该实例的工单 | `磁盘使用率超过90%` |
| **匹配规则** (rule_id) | 命中的自愈规则 | `磁盘空间告警自愈规则` |
| **状态** (status) | 当前实例状态 | `completed` |
| **当前节点** (current_node_id) | 正在执行的节点 | `execution-cleanup` |
| **开始时间** (started_at) | 实例启动时间 | `2024-01-15 10:30:00` |
| **结束时间** (completed_at) | 实例完成时间 | `2024-01-15 10:35:22` |
| **错误信息** (error_message) | 失败时的错误描述 | `SSH connection refused` |

### 3.2 节点状态表 (node_states)

实例的 `node_states` 字段记录了每个节点的运行状态，是一个 JSON 对象：

```json
{
  "start": {"status": "success", "started_at": "...", "completed_at": "..."},
  "host_extractor": {"status": "success", "output": {"hosts": ["10.0.1.1"]}},
  "cmdb_validator": {"status": "success", "output": {"valid_hosts": ["10.0.1.1"]}},
  "execution_cleanup": {"status": "success", "execution_run_id": "..."},
  "notification": {"status": "success"},
  "end": {"status": "success"}
}
```

### 3.3 流程上下文 (context)

记录流程运行过程中积累的所有变量数据，可用于问题排查：

```json
{
  "incident": {"title": "磁盘使用率超过90%", "severity": "high", ...},
  "matched_rule": {"name": "磁盘空间告警自愈规则"},
  "extracted_hosts": ["10.0.1.1"],
  "validated_hosts": ["10.0.1.1"],
  "execution_result": {"status": "success", "ok": 3, "changed": 1}
}
```

## 四、实例详情页

### 4.1 流程可视化回放

进入实例详情后，页面会展示流程的 DAG 图，每个节点根据其状态动态着色：
- 🟢 绿色 = 成功
- 🔴 红色 = 失败
- 🔵 蓝色 = 正在运行（有动画效果）
- 🟠 橙色 = 等待审批
- ⚪ 灰色 = 未执行/已跳过

### 4.2 SSE 实时推送

运行中的实例通过 **Server-Sent Events (SSE)** 实时推送节点状态变更和日志。SSE 事件类型：

| 事件类型 | 说明 |
|---------|------|
| `flow_start` | 流程实例开始执行 |
| `node_start` | 某个节点开始执行 |
| `node_log` | 节点产生日志输出 |
| `node_complete` | 某个节点执行完成 |
| `flow_complete` | 流程实例执行完毕 |

### 4.3 执行日志面板

每个节点的执行日志按时间排列，包括日志级别：

| 级别 | 颜色 | 说明 |
|------|------|------|
| `debug` | 灰色 | 调试信息 |
| `info` | 蓝色 | 一般信息 |
| `warn` | 黄色 | 警告信息 |
| `error` | 红色 | 错误信息 |

## 五、核心操作

### 5.1 查看实例详情
在列表中点击实例可进入详情页，查看可视化流程图和节点状态。

### 5.2 处理审批
对于 `waiting_approval` 状态的实例，审批人可以：
- 点击 **批准** 让流程继续
- 点击 **拒绝** 并填写原因
- 等待超时自动处理

### 5.3 取消实例
对于 `running` 状态的实例，可点击 **取消** 终止执行。

### 5.4 筛选与搜索
- 按状态筛选
- 按流程名称搜索
- 按关联工单搜索
- 按时间范围查询

---

## 六、最佳实践

1. **实时监控**：对于正在运行的自愈实例，通过 SSE 实时查看执行进展。
2. **失败分析**：实例失败时，先查看 `error_message` 和 `node_states` 定位出错的节点，再检查该节点的详细日志。
3. **上下文排查**：流程上下文 (context) 记录了各步骤的输入输出，是排查数据流问题的关键。
4. **审批及时处理**：`waiting_approval` 的实例会阻塞流程执行，建议配置通知提醒审批人。
