# 插件管理 (Plugins) 功能指南

![插件管理列表](/guides/images/plugins_overview.webp)

插件是 Auto Healing 平台与外部系统之间的**数据桥梁**。通过安装和配置不同类型的插件，平台可以从 Zabbix、Prometheus、ServiceNow 等外部系统自动拉取告警工单和资产信息，实现数据源的灵活接入。

---

## 一、插件类型

| 类型 | 代码标识 | 说明 | 典型集成系统 |
|------|---------|------|-------------|
| **ITSM** | `itsm` | IT 服务管理 / 告警工单类插件 | Zabbix、Prometheus、ServiceNow、PagerDuty |
| **CMDB** | `cmdb` | 配置管理数据库类插件 | 外部 CMDB 系统、云平台 API |

---

## 二、插件状态 (status) 字典

| 枚举值 | 中文含义 | 颜色标识 | 说明 |
|--------|---------|---------|------|
| `active` | **运行中** | 🟢 绿色 | 插件正常运行，定时同步开启 |
| `inactive` | **未激活** | ⚪ 灰色 | 插件已安装但未启用，不会执行任何同步 |
| `error` | **异常** | 🔴 红色 | 插件运行出错（如 API 认证失败、网络不通），需要排查 |

### 自动暂停机制
当插件连续同步失败达到 `max_failures`（默认 5 次）阈值时，平台会**自动将插件暂停**并记录暂停原因。管理员修复问题后需手动重新启用。

---

## 三、同步日志状态字典

每次同步操作都会产生一条同步日志，其状态如下：

| 枚举值 | 中文含义 | 说明 |
|--------|---------|------|
| `running` | **同步中** | 正在从外部系统拉取数据 |
| `success` | **成功** | 同步完成，数据已更新 ✅ |
| `failed` | **失败** | 同步过程出错 ❌ |

### 同步触发方式 (sync_type)

| 枚举值 | 中文含义 | 说明 |
|--------|---------|------|
| `scheduled` | **定时** | 按配置的间隔自动触发 |
| `manual` | **手动** | 管理员手动点击同步按钮触发 |
| `webhook` | **Webhook** | 外部系统主动推送触发 |

---

## 四、字段详解

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **名称** (name) | 插件的唯一标识名称 | `Zabbix-Production` |
| **类型** (type) | 插件类型 | `itsm` / `cmdb` |
| **版本** (version) | 插件版本 | `1.0.0` |
| **状态** (status) | 当前运行状态 | `active` / `inactive` / `error` |
| **同步开关** (sync_enabled) | 是否启用定时同步 | `true` / `false` |
| **同步间隔** (sync_interval_minutes) | 定时同步的间隔时间（分钟） | `5` |
| **上次同步** (last_sync_at) | 最后一次成功同步的时间 | `2024-01-15 10:30:00` |
| **下次同步** (next_sync_at) | 下一次计划同步的时间 | `2024-01-15 10:35:00` |
| **连续失败** (consecutive_failures) | 当前连续同步失败的次数 | `0` |
| **最大容忍** (max_failures) | 连续失败多少次后自动暂停 | `5` |

## 五、核心操作

### 5.1 安装新插件
1. 点击 **+ 新建** 按钮。
2. 填写插件名称、选择类型（ITSM/CMDB）。
3. 配置 API 连接信息：
   - **API 地址**：外部系统的 REST API 端点
   - **认证方式**：Token / Basic Auth / OAuth2
   - **认证凭据**：API Key 或用户名密码
4. 配置**字段映射**：将外部系统的字段名映射为平台标准字段（如 `zabbix_trigger_name` → `title`）。
5. 设置同步间隔和过滤规则。
6. 保存后插件默认为 `inactive` 状态，需手动启用。

### 5.2 配置字段映射
字段映射决定了外部告警数据如何转换为平台标准的工单格式。映射配置是 JSON 格式：

```json
{
  "title": "$.trigger_name",
  "severity": "$.priority",
  "affected_ci": "$.host.name",
  "description": "$.trigger_description"
}
```

### 5.3 配置同步过滤器
可设置过滤规则，只同步满足条件的数据。例如：只同步 severity >= high 的告警。

### 5.4 查看同步日志
在操作栏点击 **日志** 图标，可查看该插件的历史同步记录，包括每次同步的：
- 拉取数量 (records_fetched)
- 过滤数量 (records_filtered)
- 实际入库数量 (records_processed)
- 新增 / 更新 / 失败数量

### 5.5 手动触发同步
点击 **同步** 按钮可立即触发一次同步，无需等待定时周期。

### 5.6 停用与删除
- **停用**：将状态设为 `inactive`，停止所有同步操作。
- **删除**：删除插件配置。**已产生的工单数据不会被级联删除**，但会失去来源追溯。

---

## 六、最佳实践

1. **先测试再启用**：新插件创建后，先手动触发一次同步，确认字段映射和数据格式正确后再启用定时同步。
2. **合理设置间隔**：高频告警源建议 1-5 分钟，低频变更源可设置 15-60 分钟。
3. **监控同步健康度**：定期检查同步日志中 `records_failed` 的数量，如持续 > 0 说明映射配置可能有问题。
4. **利用过滤器降噪**：通过同步过滤器排除信息类（info）告警，避免工单表膨胀。
