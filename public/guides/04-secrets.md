# 密钥管理 (Secrets) 功能指南

![密钥列表](/guides/images/secrets_overview.webp)

密钥管理是 Auto Healing 平台的**统一凭证保险库**。所有需要连接目标主机的操作（Playbook 执行、连通性检测等），都需要通过密钥管理中配置的凭证来完成认证。密钥信息采用加密存储，确保敏感数据安全。

---

## 一、核心概念

### 1.1 密钥源 (Secret Source)
密钥源是一组认证凭据的集合。一个密钥源可以包含 SSH 私钥、用户名密码、或 API Token 等不同类型的认证信息。每个密钥源可以绑定到一组或多组 CMDB 主机。

### 1.2 优先级 (Priority)
当同一台主机命中多个密钥源时，平台会按照**优先级从高到低**逐个尝试连接，直到认证成功。数字越小优先级越高（如 `1` > `2` > `3`）。

### 1.3 Lookup Key
用于在自愈流程中通过表达式动态引用密钥源的唯一标识符。例如 `{{ secrets.prod_ssh_key }}`。

---

## 二、字段详解

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **名称** (name) | 密钥源的描述性名称 | `生产环境 SSH 密钥` |
| **类型** (type) | 凭证类型 | `SSH 密钥` / `用户名密码` / `API Token` |
| **认证方式** (auth_method) | 具体的认证手段 | `private_key` / `password` |
| **优先级** (priority) | 匹配优先级（数字越小越优先） | `1` |
| **状态** (status) | 是否启用 | `active`（启用）/ `inactive`（停用）|
| **Lookup Key** | 表达式引用标识符 | `prod_ssh_key` |
| **绑定主机数** | 该密钥源关联的 CMDB 主机数量 | `25` |
| **最后使用** | 上次被用于认证的时间 | `5 分钟前` |

---

## 三、核心操作

### 3.1 新建密钥源

1. 点击 **+ 新建** 按钮打开创建表单。
2. 填写基本信息：名称、Lookup Key（英文标识符）、优先级。
3. 选择认证类型和方式：
   - **SSH 私钥**：粘贴 PEM 格式的私钥内容，可选输入 passphrase。
   - **用户名密码**：填写用于 SSH 登录的用户名和密码。
   - **API Token**：用于调用外部 API 的认证令牌。
4. 点击 **确定** 保存。密钥内容会被加密后存入数据库。

> ⚠️ **安全提示：** 私钥和密码仅在创建时可见，保存后无法回显。如需更换，请重新填写。

### 3.2 编辑与删除
- **编辑**：可修改名称、优先级、Lookup Key，但凭证内容需重新输入才会更新。
- **删除**：已被任务模板引用的密钥源**不可直接删除**，需先解除引用。

### 3.3 绑定到 CMDB 主机
在密钥源详情中，可配置该密钥源适用的目标主机范围。支持按主机分组批量绑定。

### 3.4 搜索与筛选
- 按名称、类型、Lookup Key 搜索
- 按状态筛选（启用/停用）

---

## 四、密钥选择策略

当执行任务时，平台按以下逻辑选择密钥：

1. 查找目标主机已绑定的所有 `active` 密钥源。
2. 按优先级排序（数字小的在前）。
3. 依次尝试认证，第一个成功的密钥即被使用。
4. 如果所有密钥都失败，任务报错并记录失败原因。

---

## 五、最佳实践

1. **环境隔离**：为生产、测试环境分别建立不同的密钥源，设置不同的 Lookup Key。
2. **最小权限**：使用专门的运维账号创建密钥，避免使用 root 账号。
3. **定期轮转**：建议每 90 天更新一次 SSH 密钥和密码。
4. **优先级规划**：常用密钥设为高优先级（如 1），备用密钥设为低优先级（如 10）。
