# 主机资产管理

纳管主机、分组管理和状态监控。

## CMDB 概述

CMDB（配置管理数据库）是平台的基础设施层，管理所有纳管主机的生命周期。主机状态分为三种：

- 🟢 **在线** — 主机可达，可执行任务
- 🔴 **离线** — 主机不可达，任务将跳过
- 🟡 **维护中** — 主机处于维护模式，暂停自动任务执行

所有自愈任务和定时任务的执行都依赖 CMDB 中的主机清单。

## 主机添加方式

### 手动添加

在资产管理页面点击「添加主机」，填写主机 IP 地址、SSH 端口（默认 22）、选择认证密钥。添加后平台自动检测连通性。

### 批量导入

支持 CSV 格式批量导入主机列表，格式为：

```
IP,端口,主机名,分组
192.168.1.10,22,web-01,web-servers
192.168.1.11,22,web-02,web-servers
```

**注意事项**：

- 确保平台服务器到目标主机的网络连通
- SSH 密钥需要提前在密钥管理中配置
- 建议为不同环境使用不同的密钥

## 分组与标签

**主机分组**用于逻辑隔离和批量管理：

- 按环境分组：production、staging、development
- 按业务分组：web-servers、db-servers、cache-servers
- 按地域分组：beijing、shanghai、hongkong

分组支持嵌套，便于层级管理。在任务模板中选择主机时，可以按分组批量选择，避免逐台指定。
