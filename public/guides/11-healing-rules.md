# 自愈规则 (Healing Rules) 功能指南

![自愈规则列表](/guides/images/healing_rules_overview.webp)

自愈规则是告警事件与自愈流程之间的**匹配引擎**。每条规则定义了一组条件表达式，当新工单到达时，自愈引擎会按优先级逐条匹配规则，首个命中的规则将触发其关联的自愈流程。

---

## 一、核心概念

### 1.1 匹配流程
```
新工单到达 → 自愈引擎按优先级逐条扫描规则 
→ 条件匹配？
  ├─ 是 → 检查触发模式
  │   ├─ auto → 自动触发流程 (healing_status=processing)
  │   └─ manual → 进入待办中心等待审批
  └─ 否 → 继续下一条规则
       └─ 所有规则都不匹配 → healing_status=skipped
```

### 1.2 规则优先级
数字越小优先级越高。一旦某条规则匹配成功，后续规则**不再继续匹配**。这意味着：
- 设置 `priority=1` 的规则会最先被检查。
- 更具体的规则应设置更高的优先级。
- 通用兜底规则应设最低优先级。

---

## 二、字段详解

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **名称** (name) | 规则名称 | `磁盘空间告警自愈规则` |
| **描述** (description) | 规则用途说明 | `匹配磁盘空间不足的告警，自动清理临时文件` |
| **优先级** (priority) | 匹配优先级，越小越先匹配 | `1` |
| **启用** (is_active) | 是否启用 | `true` / `false` |
| **关联流程** (flow_id) | 匹配后触发的自愈流程 | `磁盘空间自愈流程` |
| **上次执行** (last_run_at) | 上次成功匹配并触发的时间 | `5 分钟前` |

---

## 三、触发模式 (trigger_mode) 字典

| 枚举值 | 中文含义 | 说明 |
|--------|---------|------|
| `auto` | **自动触发** | 规则匹配后**立即自动启动**自愈流程，无需人工确认 |
| `manual` | **手动触发** | 规则匹配后，工单进入**待办中心**，需运维人员在待触发工单页面手动确认后才启动流程 |

> **选择建议：**
> - 风险低的操作（如日志清理、重启服务）→ `auto`
> - 风险高的操作（如数据库修复、配置变更）→ `manual`

---

## 四、匹配模式 (match_mode) 字典

| 枚举值 | 中文含义 | 说明 |
|--------|---------|------|
| `all` | **全部满足** | 所有条件都必须为真（AND 逻辑） |
| `any` | **任一满足** | 任意一个条件为真即匹配（OR 逻辑） |

---

## 五、条件表达式

### 5.1 条件结构
每条条件由三部分组成：**字段** + **操作符** + **值**

### 5.2 支持的操作符 (operator) 字典

| 枚举值 | 中文含义 | 说明 | 示例 |
|--------|---------|------|------|
| `equals` | **等于** | 完全匹配 | `severity equals "critical"` |
| `contains` | **包含** | 子串匹配（不区分大小写） | `title contains "磁盘"` |
| `in` | **在列表中** | 值在给定的列表中 | `status in ["open","in_progress"]` |
| `regex` | **正则匹配** | 正则表达式匹配 | `title regex "disk.*full"` |
| `gt` | **大于** | 数值比较 | `priority gt 3` |
| `lt` | **小于** | 数值比较 | `priority lt 2` |
| `gte` | **大于等于** | 数值比较 | `priority gte 1` |
| `lte` | **小于等于** | 数值比较 | `priority lte 5` |

### 5.3 条件组
支持嵌套的条件组（Group），实现复杂的逻辑组合：
- 组内条件使用 `AND` 或 `OR` 连接
- 组与组之间可以再次组合

例如：
```
(severity = "critical" AND affected_ci contains "db")
OR
(severity = "high" AND category = "database")
```

### 5.4 可用字段
条件表达式可以引用工单的以下字段：

| 字段路径 | 说明 |
|---------|------|
| `title` | 工单标题 |
| `description` | 工单描述 |
| `severity` | 告警级别 |
| `priority` | 优先级 |
| `status` | 工单状态 |
| `category` | 分类 |
| `affected_ci` | 受影响 CI |
| `affected_service` | 受影响服务 |
| `source_plugin_name` | 来源插件 |
| `raw_data.xxx` | 原始数据中的任意字段（支持嵌套路径） |

## 六、核心操作

### 6.1 创建规则
1. 点击 **+ 新建**，填写名称和描述。
2. 设置优先级和触发模式。
3. 选择匹配模式（全部/任一）。
4. 添加条件表达式：选择字段、操作符、填写值。
5. 选择关联的自愈流程。
6. 可选：是否立即启用。

### 6.2 启用/停用
通过开关控制规则是否参与匹配。停用的规则在自愈扫描时会被跳过。

### 6.3 测试规则
可使用**模拟匹配**功能输入一条测试工单数据，验证规则是否能正确匹配。

---

## 七、最佳实践

1. **优先级规划**：建议预留间隔设置优先级（如 10, 20, 30），方便后续插入新规则。
2. **精确匹配优先**：越具体的条件越应设置高优先级，通用规则放最后。
3. **先 manual 后 auto**：新规则初期建议使用 `manual` 模式观察效果，确认稳定后切换为 `auto`。
4. **利用 raw_data**：外部系统的原始字段（通过 `raw_data.xxx` 路径引用）往往提供更精确的匹配条件。
